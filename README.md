# PKU 课表美化与期末安排

一个用于北京大学门户课表页的 Tampermonkey 脚本。在保留原有课表信息的基础上，提供更清爽的课表视图、可编辑的期末安排以及课程数据导出能力。

## 功能

- 重构课表界面，让周一到周日、节次、单双周和课程颜色更适合宽屏阅读。
- 使用统一顶栏切换「课表 / 期末安排」「跟随系统 / 浅色 / 深色」以及吉祥物装饰的显示状态。
- 分层展示课程时间、地点、教师、期末考试时间和考试地点等信息。
- 课程格空白较多时自动加入低透明度院系吉祥物装饰。
- 自动生成可编辑的期末安排时间轴，并支持手动设置期末考试、随堂考试、大作业和论文等结课方式。
- 支持将期末安排导出为 PNG 图片。
- 支持导出 Wakeup 课程表可导入的 CSV 文件。
- 门户课程响应只在当前页面内存中处理；手动补充信息和界面偏好仅保存在浏览器本地。

## 安装

目前支持 Chromium 内核浏览器，推荐使用 Edge 或 Chrome。

1. 安装 [Tampermonkey（篡改猴）](https://www.tampermonkey.net/)，选择黑色图标的正式版本。
2. 打开 [Greasy Fork 上的“PKU 课表美化与期末安排”](https://greasyfork.org/zh-CN/scripts/586285-pku-%E8%AF%BE%E8%A1%A8%E7%BE%8E%E5%8C%96%E4%B8%8E%E6%9C%9F%E6%9C%AB%E5%AE%89%E6%8E%92)并安装脚本。
3. 在浏览器扩展管理页面开启开发人员模式，然后打开 Tampermonkey 的详细信息，按下图设置权限：

   ![权限设置示意图](./img/setting-example.png)

已经安装 [PKU-ART2](https://github.com/zhuozhiyongde/PKU-Art) 的用户可以忽略第 3 步。PKU-ART2 支持教学网美化、一键下载课程内容以及录播下载与重命名。

也可以直接复制仓库根目录中的 [`pku-exam-timeline.user.js`](./pku-exam-timeline.user.js) 到 Tampermonkey 中安装。

## 使用

1. 打开 `https://portal.pku.edu.cn/publicQuery/#/myCourseTable`，即门户网站中的“我的课表”。
2. 第一次使用时刷新页面或切换学期/周次，让课表重新加载；以后打开课表页会自动读取数据。
3. 使用顶栏切换课表、期末安排、颜色模式和吉祥物装饰。
4. 在期末安排页补全缺失的地点、日期和结课方式等信息。
5. 点击「导出图片」保存期末安排时间轴和 DDL。
6. 点击「导出 CSV」生成 Wakeup 可导入的课程表文件。

## Wakeup CSV

导出的 CSV 字段与 Wakeup 模板保持一致：

```text
课程名称,星期,开始节数,结束节数,老师,地点,周数
```

- 星期使用 `1-7`。
- 连续节次会合并为一条记录。
- 字段内半角逗号会转换为中文顿号，减少导入失败。
- 单独某一周不会追加单双周后缀。
- 空字段会填入 `无`。

导入时，将 CSV 保存到手机，通过 QQ“我的电脑”或微信文件传输助手发送，然后使用 Wakeup 打开该文件即可。

## 隐私

脚本不保存账号密码，不读取或上传 Cookie。门户课程响应不会持久化；手动编辑的补充信息和界面偏好保存在当前浏览器的 `localStorage` 中。

## 兼容性

- 推荐使用 Edge 或 Chrome。
- 需要 Tampermonkey。
- 仅适配北京大学门户课表页。

## 开发

这是一个纯 userscript 项目，没有构建步骤。修改后可进行基本语法检查：

```bash
node --check pku-exam-timeline.user.js
```

吉祥物源 PNG 保存在本地 `assets/doodles-raster` 目录，并通过根目录 `.gitignore` 排除，不上传到 GitHub；运行时图片以内嵌 Base64 的形式保存在 userscript 中。更新素材后，在仓库根目录执行：

```bash
# 检查源 PNG、内嵌 Base64、素材映射和 userscript 语法
node scripts/sync-doodles.mjs --check

# 将本地源 PNG 重新嵌入 userscript，并再次完成上述检查
node scripts/sync-doodles.mjs --write
```

同步脚本会检查 PNG 格式、透明通道、最长边不超过 320px、单图不超过 64 KiB、内容不重复，以及预览页中的文件名与内部标识是否一致，避免手工替换 Base64 时出现漏图、错图或语法错误。
